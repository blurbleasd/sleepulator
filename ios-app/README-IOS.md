# Sleepulator iOS Native App

This directory contains the Swift/SwiftUI source code for the native iOS Sleepulator app, transitioning away from the web app platform to guarantee rock-solid background audio and native performance.

## Setup Instructions

Since Xcode manages project files (`.xcodeproj`) via its GUI, you will need to create the Xcode project and drag these files in. It takes about 2 minutes.

1. **Create the Project**:
   - Open Xcode and go to **File → New → Project**.
   - Select **iOS → App** and click Next.
   - **Product Name**: `Sleepulator`
   - **Interface**: SwiftUI
   - **Language**: Swift
   - Click Next and save it anywhere (you can save it right in this repository folder).

2. **Import the Code**:
   - In Xcode's left sidebar (Project Navigator), delete the auto-generated `ContentView.swift`.
   - Open Finder, navigate to this `ios-app/` folder, and drag the `Models`, `Services`, and `Views` folders directly into your Xcode project sidebar. 
   - Ensure **"Copy items if needed"** is *unchecked* (so you are editing these exact files in the repo), and **"Create groups"** is selected.

3. **Enable Background Audio**:
   - Click on the root `Sleepulator` project file in the sidebar.
   - Go to **Signing & Capabilities**.
   - Click the **+ Capability** button in the top left.
   - Double-click **Background Modes**.
   - Check **"Audio, AirPlay, and Picture in Picture"**. (If you skip this step, audio will stop when you lock your screen).

4. **Update App Entry Point**:
   - Open your `SleepulatorApp.swift` (auto-generated by Xcode).
   - Change the body to point to our new root view:
     ```swift
     var body: some Scene {
         WindowGroup {
             ContentView()
         }
     }
     ```

5. **Run the App**:
   - Select your physical iPhone from the device dropdown at the top.
   - Hit **⌘R** to build and run.

## What's Included

- **`HomeView.swift`**: A completely redesigned, highly intuitive home layout. It focuses entirely on mixing your audio sources with a giant master play/pause button and sleek sliders.
- **`LibraryView.swift`**: Replaces the podcast screen. It features a native `XMLParser` (`PodcastParser.swift`) to fetch RSS feeds directly without relying on a proxy. Long podcast titles are natively truncated with `.lineLimit(1)`, fixing the "wide podcast name" bug.
- **`AudioEngine.swift`**: The core AVAudioEngine from your spike, now wired into the new UI layout.
